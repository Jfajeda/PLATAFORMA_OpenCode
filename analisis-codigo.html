1. Crear cuenta en SonarCloud
  1. Ve a sonarcloud.io e inicia sesion con GitHub
  2. Crea una organizacion (o usa la existente: jfajeda)
  3. Importa tu repositorio como proyecto nuevo
  4. Genera un token en: My Account > Security > Generate Token
  5. Guarda el token como variable de entorno: 
    export SONAR_TOKEN="tu-token-aqui"
2. Configurar sonar-project.properties

Crea este archivo en la raiz de tu proyecto:

sonar.projectKey=Jfajeda_PLATAFORMA_OpenCode
sonar.organization=jfajeda
sonar.projectName=Plataforma OpenCode
sonar.sources=.
sonar.sourceEncoding=UTF-8
sonar.exclusions=**/node_modules/**,**/dist/**,**/.git/**
sonar.qualitygate.wait=true
3. Ejecutar analisis
# Con npx (sin instalacion previa)
export SONAR_TOKEN="tu-token"
npx @sonar/scan

# O instalar globalmente
npm install -g @sonar/scan
sonar
4. Integrar MCP en OpenCode

Anade esto a tu opencode.json para que OpenCode verifique codigo automaticamente:

{
  "mcp": {
    "sonarqube": {
      "type": "local",
      "command": ["npx", "-y", "sonarqube-mcp-server@latest"],
      "environment": {
        "SONARQUBE_URL": "https://sonarcloud.io",
        "SONARQUBE_TOKEN": "{env:SONAR_TOKEN}",
        "SONARQUBE_ORGANIZATION": "jfajeda"
      }
    }
  }
}
Comando custom /sonar Si has creado el archivo .opencode/commands/sonar.md, puedes ejecutar el analisis directamente desde OpenCode escribiendo /sonar.
5. Exportar este panel a otro proyecto
  1. Copia el archivo analisis-codigo.html a tu otro proyecto
  2. Abre el archivo y edita el bloque SONAR_CONFIG al inicio del <script>
  3. Cambia organization, projects y brand segun necesites
  4. Listo. El panel funciona de forma independiente
Token de API Si dejas el token vacio, el panel mostrara badges publicos (solo para proyectos publicos en SonarCloud). Para proyectos privados necesitas el token.
